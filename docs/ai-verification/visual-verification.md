# UI/렌더링 변경 시 검증 규칙

스크린샷 이미지의 "눈으로 보이는지"에 의존하지 말고, 코드 레벨에서 논리적으로 검증할 것.

## 반드시 확인할 체크리스트

### 1. 색상 대비

새로운 UI 요소를 그릴 때, 해당 요소의 색상과 배경색의 RGB 차이를 계산한다.

- 배경색과 동일하거나 유사한 색으로 그리면 **확대하든 말든 보이지 않는다**.
- 최소 대비: 각 채널 (R, G, B) 중 하나 이상이 40 이상 차이나야 한다.

```
예: 배경 rgb(26, 26, 30)에 경계선 rgb(60, 60, 75)
→ 차이: (34, 34, 45) — 최대 45로 겨우 보임, 더 밝게 해야 함
→ 최소 rgb(90, 90, 110) 이상을 사용할 것
```

### 2. 렌더링 레이어/순서

요소가 실제로 화면에 나타나는지 렌더 파이프라인의 순서를 코드에서 추적한다.

- egui `LayerId::background()`는 모든 패널 **뒤**에 그려진다 → 터미널 위에 그리려면 `Order::Foreground` 사용
- wgpu `queue.write_buffer()`는 `queue.submit()` 시점의 **마지막 값**만 유효하다 → 여러 surface를 하나의 encoder에 넣으면 마지막 것만 렌더링됨

### 3. 픽셀 수치 검증

스크린샷을 찍은 후 반드시 해당 영역의 RGB 값을 직접 읽어서 기대 값과 비교한다.

```python
# 경계선이 있어야 할 위치의 픽셀을 확인
for x in range(expected_x - 5, expected_x + 5):
    idx = (y * width + x) * 3
    r, g, b = data[idx], data[idx+1], data[idx+2]
    print(f'pixel({x},{y}) = rgb({r},{g},{b})')
```

기대하는 색상이 해당 좌표에 실제로 존재하는지 확인한 후에야 "보인다"고 판단할 수 있다.

### 4. "보인다"고 주장하기 전에

- [ ] 해당 색상이 배경색과 다른가? (코드에서 확인)
- [ ] 해당 레이어가 다른 요소 위에 렌더링되는가? (렌더 순서 확인)
- [ ] 해당 영역의 픽셀 RGB 값이 기대값과 일치하는가? (수치 확인)

세 가지 모두 통과하지 않으면 "보인다"고 말하지 않는다.
