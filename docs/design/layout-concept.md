# 두 레벨 레이아웃 — Tasty의 핵심 설계

## 문제

기존 터미널 에뮬레이터/멀티플렉서는 화면 분할에 대해 하나의 정책만 제공한다:

- **tmux**: 분할(pane)은 window에 고정. 탭(window)을 전환해도 분할은 유지됨. 탭별로 다른 분할 배치를 가질 수 없음.
- **iTerm2**: 분할(pane)은 tab에 종속. 탭을 전환하면 분할도 바뀜. 탭과 무관한 고정 분할이 불가.

## 해결

Tasty는 **상위 레이아웃**과 **하위 레이아웃** 두 단계를 제공한다.

### 상위 레이아웃 (탭과 무관)

워크스페이스 내에서 Pane Group들을 배치한다. 이 분할은 탭을 전환해도 변하지 않는다.

**용도**: 화면의 물리적 영역을 나눠서, 각 영역이 독립적으로 탭을 전환할 수 있게 한다.

```
┌────────────────────┬──────────────┐
│  Pane Group A      │ Pane Group B │
│  [Tab1] [Tab2]     │ [Tab1]       │
│                    │              │
│  (탭 전환해도      │ (독립적으로  │
│   이 분할 유지)    │  탭 전환)    │
└────────────────────┴──────────────┘
```

### 하위 레이아웃 (탭에 종속)

Pane(탭) 내에서 Surface(터미널)들을 배치한다. 탭을 전환하면 이 분할도 함께 전환된다.

**용도**: 하나의 탭 안에서 여러 터미널을 동시에 보면서 작업한다. 다른 탭으로 전환하면 다른 배치가 표시된다.

```
Pane Group A의 Tab 1:          Pane Group A의 Tab 2:
┌──────────┬──────────┐        ┌─────────────────────┐
│ Surface1 │ Surface2 │   →    │     Surface3        │
│          │          │  전환   │                     │
│          ├──────────┤        │                     │
│          │ Surface3 │        │                     │
└──────────┴──────────┘        └─────────────────────┘
```

## 사용 예시

### Claude Code + 참조 터미널

상위 레이아웃으로 좌우 분할. 왼쪽은 Claude Code 전용, 오른쪽은 참조용.

```
┌──────────────────┬─────────────────┐
│  Pane Group L    │  Pane Group R   │
│  [Claude]        │  [logs] [build] │
│                  │                 │
│  claude code     │  tail -f logs   │
│  실행 중         │  (탭 전환 가능) │
└──────────────────┴─────────────────┘
```

오른쪽 Pane Group에서 탭을 전환해도 왼쪽의 Claude Code는 영향 없음.

### 멀티 서비스 개발

하위 레이아웃으로 탭별 분할 구성:

- Tab 1 "Frontend": Surface 3개 (dev server, test, terminal)
- Tab 2 "Backend": Surface 2개 (server, database)
- Tab 3 "Deploy": Surface 1개

탭 전환으로 작업 컨텍스트를 빠르게 전환. 각 탭의 분할 배치는 독립적으로 유지.
