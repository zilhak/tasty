# 09. 명령 팔레트

## cmux 구현 방식

- Cmd+Shift+P로 열기
- 퍼지 매칭 검색
- 비동기 검색 (타이핑 지연 없음)
- 워크스페이스 이름변경 통합
- 전체 윈도우 스위처

## 크로스 플랫폼 구현 방안

### 구현 방법

VS Code 스타일의 명령 팔레트. egui 오버레이 위젯으로 구현한다.

```
┌──────────────────────────────────┐
│ > search query_                  │
├──────────────────────────────────┤
│ 🔧 New Workspace          Ctrl+N │
│ ▸ Split Horizontal    Ctrl+Sh+D │
│   Split Vertical         Ctrl+D │
│   Rename Workspace              │
│   Close Pane             Ctrl+W │
│ ⚙ Settings          Ctrl+Comma │
│ 🔍 Find in Terminal      Ctrl+F │
└──────────────────────────────────┘
```

### 핵심 요소

| 요소 | 구현 |
|------|------|
| 퍼지 매칭 | `nucleo` 크레이트 (fzf 알고리즘, 빠름) |
| 오버레이 렌더링 | egui로 반투명 배경 + 패널을 화면 중앙에 렌더링 |
| 명령 등록 | 각 기능이 명령 팔레트에 항목을 등록하는 레지스트리 패턴 |
| 키 바인딩 표시 | 각 명령 옆에 바인딩된 단축키 표시 |
| 카테고리 필터 | `>` 명령, `@` 워크스페이스, `#` 설정 등 접두사 필터 |
| 입력 필드 | IME 지원 텍스트 입력 (winit IME 이벤트) |
| 애니메이션 | 열기/닫기 시 부드러운 페이드/슬라이드 |

### 명령 팔레트 모드

| 접두사 | 모드 | 설명 |
|--------|------|------|
| (없음) | 명령 검색 | 모든 명령을 퍼지 검색 |
| `>` | 명령 전용 | 명령만 표시 |
| `@` | 워크스페이스 전환 | 워크스페이스 목록에서 전환 |
| `#` | 설정 | 설정 항목 검색 및 변경 |

### 참고

- VS Code의 Command Palette UX가 목표
- WezTerm의 명령 팔레트도 참고 (Launcher)

## 최적화 전략

- **지연 초기화**: 명령 팔레트를 처음 열 때만 명령 목록을 수집한다. 앱 시작 시에는 로드하지 않는다.
- **검색 디바운스**: 키 입력마다 검색하지 않고 50~100ms 디바운스를 적용한다. 빠르게 타이핑할 때 불필요한 매칭 연산을 줄인다.
- **퍼지 매칭 최적화**: `nucleo` 크레이트의 병렬 매칭을 활용하고, 결과 개수를 제한(예: 상위 50개)하여 대량 명령 목록에서도 빠르게 응답한다.
- **렌더링**: 보이는 결과 항목만 렌더링한다(가상 스크롤). 수백 개의 매칭 결과가 있어도 화면에 보이는 10~15개만 그린다.
