# 15. 원격 SSH

## cmux 구현 방식

- Go 기반 원격 데몬 (cmuxd-remote)
- SHA-256 검증된 데몬 자동 배포
- SOCKS5/HTTP CONNECT 프록시 터널링
- CLI 릴레이 (원격에서 로컬 cmux 명령 실행)
- 역방향 SSH 포트 포워딩

## 크로스 플랫폼 구현 방안

### 현실적 평가

SSH 원격 세션은 복잡한 기능이다. MVP 이후 확장 기능으로 분류.

### 구현 방식

| 컴포넌트 | 방법 |
|---------|------|
| SSH 클라이언트 | `russh` 크레이트 (pure Rust SSH 구현) |
| 원격 데몬 | Rust로 작성, 크로스 컴파일하여 원격에 배포 |
| 프록시 터널 | SOCKS5 프록시를 SSH 채널 위에 구현 |

### OS별 고려사항

| OS | SSH 클라이언트 | 원격 데몬 타겟 |
|----|--------------|--------------|
| **Windows** | `russh` (순수 Rust) | Linux/macOS 서버에 배포 |
| **macOS** | `russh` 또는 시스템 ssh | Linux/macOS 서버에 배포 |
| **Linux** | `russh` 또는 시스템 ssh | 동일 또는 원격 Linux |

### 원격 데몬 배포

Rust로 원격 데몬을 작성하면 크로스 컴파일로 여러 타겟을 지원:
- `x86_64-unknown-linux-gnu`
- `aarch64-unknown-linux-gnu`
- `x86_64-apple-darwin`
- `aarch64-apple-darwin`

### 대안: 시스템 ssh 활용

원격 데몬 없이 시스템 `ssh` 명령을 직접 호출하는 경량 방식도 가능.
이 경우 PTY를 SSH를 통해 원격 셸에 연결만 하면 된다.

### GUI에서의 SSH 관리

GUI 앱이므로 SSH 연결 관리 UI를 제공할 수 있다:
- SSH 호스트 목록 (설정에서 관리)
- 연결 상태 표시
- 키 인증 / 비밀번호 입력 대화상자
- 포트 포워딩 관리 패널

## 최적화 전략

- **연결 멀티플렉싱**: 동일 호스트에 대한 여러 세션을 하나의 SSH 연결로 공유한다. `russh`의 채널 멀티플렉싱을 활용하여 연결 설정 비용을 줄인다.
- **데이터 압축**: SSH 채널 위에 zstd 압축을 적용한다. 대량 출력(빌드 로그 등)에서 대역폭을 절약한다.
- **재연결 백오프**: 지수 백오프로 재연결을 시도하여 네트워크 부하를 방지한다. 1초 → 2초 → 4초 → 최대 30초로 간격을 늘린다.
- **로컬 에코**: 네트워크 지연이 높을 때 입력을 로컬에서 먼저 표시한다. 서버 응답이 오면 보정하여 체감 지연을 줄인다.

| OS | 가능 여부 | 비고 |
|----|-----------|------|
| Windows | ✅ 가능 | russh 사용, OpenSSH도 기본 내장 |
| macOS | ✅ 가능 | |
| Linux | ✅ 가능 | |

기능 자체는 가능하지만 복잡도가 높아 후순위로 분류.
