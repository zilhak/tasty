# 11. 검색

## cmux 구현 방식

- Cmd+F 터미널 스크롤백 검색
- 결과 카운트, 다음/이전 이동
- 브라우저 Find-in-page
- 선택 텍스트로 검색 (Cmd+E)

## 크로스 플랫폼 구현 방안

### 구현 방법

egui 검색 바 오버레이. 터미널 상단에 부드럽게 슬라이드되는 검색 패널.

```
┌──────────────────────────────────────────────┐
│ 🔍 [검색: pattern_        ] [3/17] [▲] [▼] [×] │  ← 검색 바 오버레이
├──────────────────────────────────────────────┤
│ ...                                          │
│ 여기에 [pattern] 이 있습니다                   │  ← 하이라이트 (배경색)
│ ...                                          │
│ 또 다른 [pattern] 발견                        │  ← 하이라이트 (배경색)
│ ...                                          │
└──────────────────────────────────────────────┘
```

### 핵심 요소

| 요소 | 구현 |
|------|------|
| 검색 바 | egui 오버레이, IME 지원 입력 필드 |
| 스크롤백 버퍼 | termwiz Surface에서 텍스트 추출 |
| 검색 엔진 | 단순 문자열 매칭 + 정규식 옵션 (`regex` 크레이트) |
| 하이라이트 | 매치 셀의 배경색을 변경하여 GPU 렌더링 |
| 현재 매치 | 현재 포커스된 매치는 다른 색상으로 강조 |
| 네비게이션 | Enter/Shift+Enter 또는 ▲/▼ 버튼으로 다음/이전 |
| 선택 텍스트 검색 | 현재 선택된 텍스트로 검색 필드 자동 채우기 |

### 검색 옵션

| 옵션 | 설명 |
|------|------|
| 대소문자 구분 | 토글 버튼 (Aa) |
| 정규식 | 토글 버튼 (.*) |
| 전체 단어 | 토글 버튼 (ab) |

### 스크롤백 검색 최적화

GPU 렌더링 앱이므로 검색 결과를 미니맵(스크롤바 옆 마커)으로도 표시할 수 있다.
VS Code의 검색 결과 미니맵 표시와 유사한 UX.

## 최적화 전략

- **ANSI 스트립 인덱스**: 스크롤백에 ANSI 이스케이프가 제거된 텍스트 인덱스를 별도 유지하여 검색 속도를 향상시킨다. 셀 저장 시 플레인텍스트 뷰를 함께 캐싱한다.
- **증분 검색**: 이전 검색 결과를 캐싱하고, 쿼리가 확장될 때(예: "fo" → "foo") 기존 결과 내에서만 재검색한다. 전체 스크롤백을 다시 탐색하지 않는다.
- **백그라운드 검색**: 대량 스크롤백 검색을 별도 스레드에서 처리하고 결과를 점진적으로 표시한다. 검색 중에도 터미널 입력이 블로킹되지 않는다.
- **하이라이트 최적화**: 화면에 보이는 매치만 하이라이트 렌더링한다. 스크롤백 깊숙이 있는 매치는 스크롤할 때 비로소 하이라이트를 적용한다.
