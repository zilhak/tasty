# 19. 마크다운 뷰어

## cmux 구현 방식

- SwiftUI 마크다운 렌더링
- DispatchSource 파일 감시로 라이브 리로드
- 읽기 전용 패널

## 크로스 플랫폼 구현 방안

### 구현 방법

egui 기반 리치 마크다운 렌더링. 실제 타이포그래피, 다양한 폰트 크기, 인라인 이미지로 렌더링.

### 렌더링 파이프라인

```
마크다운 텍스트
    ↓  pulldown-cmark (파싱)
AST (Abstract Syntax Tree)
    ↓  레이아웃 엔진
렌더 트리 (위치, 크기, 스타일)
    ↓  wgpu
GPU 렌더링
```

### 기능

| 기능 | 구현 |
|------|------|
| 마크다운 파싱 | `pulldown-cmark` 크레이트 |
| 텍스트 렌더링 | `cosmic-text`로 다양한 폰트 크기/스타일 |
| 코드 블록 | `syntect`로 신택스 하이라이팅, 모노스페이스 폰트 |
| 이미지 인라인 | `image` 크레이트로 디코딩 → GPU 텍스처 |
| 링크 | 클릭 가능, 호버 시 URL 표시 |
| 테이블 | GPU 렌더링 테이블 레이아웃 |
| 라이브 리로드 | `notify` 크레이트로 파일 변경 감시 |
| 스크롤 | 마우스 휠 + 키보드 스크롤, 부드러운 스크롤링 |

### 표시 영역

패인으로 열리므로 분할 패인의 한쪽에 마크다운을 표시할 수 있다:

```
┌─────────────────┬──────────────────────────┐
│  Sidebar        │  ┌──────────┬──────────┐ │
│                 │  │ Terminal │ README.md│ │
│  🟢 agent-1     │  │          │          │ │
│                 │  │  $ _     │ # Title  │ │
│                 │  │          │          │ │
│                 │  │          │ Body text│ │
│                 │  │          │ **bold** │ │
│                 │  └──────────┴──────────┘ │
└─────────────────┴──────────────────────────┘
```

### 셸 내부 폴백

터미널 패인 내에서 마크다운을 표시해야 하는 경우 (예: 셸 내부):
`termimad` 크레이트로 ANSI 텍스트 변환도 지원.

## 최적화 전략

- **점진적 렌더링**: 대용량 마크다운을 한 번에 렌더링하지 않고 보이는 영역만 렌더링한다. 스크롤에 따라 추가 영역을 점진적으로 그린다.
- **파싱 캐싱**: 파서 결과(AST)를 캐싱하고, 파일 변경 시에만 재파싱한다. `pulldown-cmark`의 파싱 결과를 메모리에 유지한다.
- **이미지 지연 로딩**: 뷰포트에 들어올 때만 이미지를 로드/디코딩한다. 스크롤되지 않은 이미지는 플레이스홀더만 표시한다.
- **코드 블록 하이라이팅 캐싱**: `syntect` 하이라이팅 결과를 캐싱한다. 동일 코드 블록에 대한 반복 하이라이팅 연산을 방지한다.
