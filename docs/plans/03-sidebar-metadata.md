# 03. 사이드바 메타데이터

## cmux 구현 방식

- Git 브랜치명 실시간 표시
- 연결된 PR 상태/번호 표시
- 작업 디렉토리 표시
- 리스닝 포트 표시
- 최신 알림 텍스트 미리보기
- 진행률 표시
- 상태 항목 (key-value, 아이콘, 색상, URL, 마크다운)

## 크로스 플랫폼 구현 방안

### 데이터 수집

| 메타데이터 | 수집 방법 | 크로스 플랫폼 |
|-----------|----------|-------------|
| Git 브랜치 | `git rev-parse --abbrev-ref HEAD` 또는 `.git/HEAD` 파일 읽기 | ✅ |
| PR 상태 | `gh pr status` 또는 GitHub API | ✅ |
| 작업 디렉토리 | PTY의 OSC 7 시퀀스 또는 OS별 프로세스 API | ⚠️ OS별 차이 |
| 리스닝 포트 | 별도 문서 참조 (14-port-scanning.md) | ⚠️ OS별 차이 |
| 알림 텍스트 | OSC 9/99/777 파싱 (termwiz에서 처리) | ✅ |

### 작업 디렉토리 추적 OS별 방법

| OS | 방법 |
|----|------|
| **Linux** | `/proc/{pid}/cwd` 심볼릭 링크 읽기 |
| **macOS** | `proc_pidinfo()` 또는 `lsof -p {pid}` |
| **Windows** | `NtQueryInformationProcess` 또는 셸의 OSC 7 시퀀스에 의존 |

termwiz에서 OSC 7을 직접 처리하므로 가장 범용적이고 정확한 방법이다.

### GUI 렌더링

네이티브 GUI이므로 실제 아이콘, 색상, 다양한 폰트 크기를 자유롭게 사용할 수 있다.

```
┌─────────────────┐
│ 🟢 workspace-1  │
│   main ← PR#42  │  ← Git 브랜치 + PR 링크
│   📁 ~/project   │  ← CWD 아이콘
│   🌐 :3000 :5173 │  ← 포트 아이콘
│   💬 "Waiting..." │  ← 알림 미리보기
│   ████░░░░ 62%   │  ← 진행률 바
├─────────────────┤
│ 🔵 workspace-2  │
│   feat/auth      │
│   📁 ~/project   │
└─────────────────┘
```

### 렌더링 요소

| 요소 | 구현 방법 |
|------|----------|
| 아이콘 | 내장 아이콘 폰트 또는 SVG 래스터라이즈 |
| 텍스트 | cosmic-text로 다양한 크기/색상/볼드 렌더링 |
| 진행률 바 | GPU 렌더링 사각형 (그라데이션 가능) |
| 색상 인디케이터 | 원형/사각형 컬러 뱃지 |
| 호버 효과 | 마우스 호버 시 상세 정보 팝오버 표시 |
| 클릭 액션 | 포트 클릭 → 브라우저 열기, PR 클릭 → GitHub 열기 |

## 최적화 전략

- **메타데이터 수집 쓰로틀링**: Git 브랜치/PR 상태 등 외부 명령 호출을 적절한 간격으로 제한한다. 예: Git 상태 2초, PR 상태 30초, 포트 스캔 5초.
- **비동기 수집**: 모든 메타데이터를 별도 스레드에서 수집하여 UI 블로킹을 방지한다. tokio 또는 OS 스레드를 사용한다.
- **변경 감지**: 수집된 값이 이전과 동일하면 사이드바 재렌더링을 스킵한다. 값이 변경될 때만 해당 영역만 다시 그린다.
- **프로세스 풀링**: 여러 워크스페이스의 메타데이터를 배치로 수집한다. 예: 모든 워크스페이스의 Git 상태를 하나의 `git` 호출로 가져오거나, 최소한 순차가 아닌 병렬로 실행한다.

## 구현 가능 여부

| OS | 가능 여부 | 비고 |
|----|-----------|------|
| Windows | ✅ 가능 | CWD 추적은 OSC 7 또는 NtQueryInformationProcess |
| macOS | ✅ 가능 | |
| Linux | ✅ 가능 | /proc 기반으로 가장 쉬움 |

GUI 앱이므로 cmux와 동등한 수준의 리치 메타데이터 표시가 가능하다.
