# 05. 분할 패인

## cmux 구현 방식

- Bonsplit 라이브러리 기반 수평/수직 분할
- 패인 간 방향키 포커스 이동
- 패인 줌/최대화 토글
- 비활성 분할 반투명화
- 드래그앤드롭 이동
- 포커스 플래시 애니메이션

## 크로스 플랫폼 구현 방안

### 레이아웃 엔진

GPU 렌더링 기반 트리 구조 분할. 각 패인은 독립적인 터미널 렌더 영역이다.

```rust
enum SplitNode {
    Leaf { pane_id: PaneId },
    Split {
        direction: Direction,  // Horizontal | Vertical
        ratio: f32,            // 0.0 ~ 1.0
        first: Box<SplitNode>,
        second: Box<SplitNode>,
    }
}
```

### 구현 요소

| 요소 | 구현 방법 |
|------|----------|
| 수평/수직 분할 | 트리 노드 삽입, GPU 렌더 영역 분할 |
| 드래그 리사이즈 | winit 마우스 이벤트로 디바이더 드래그 감지, 비율 실시간 조정 |
| 포커스 이동 | 방향키로 트리 탐색, 마우스 클릭으로 직접 포커스 |
| 줌/최대화 | 포커스된 패인만 전체 영역에 렌더링 (다른 패인 숨김) |
| 비활성 디밍 | 비활성 패인의 알파값 조정 (셰이더 기반) |
| 포커스 애니메이션 | 포커스 전환 시 테두리 글로우 애니메이션 |
| 드래그앤드롭 이동 | 패인을 드래그하여 다른 위치로 이동 |

### 디바이더 렌더링

GPU 렌더링이므로 디바이더를 시각적으로 풍부하게 표현할 수 있다:

- **기본**: 1~2px 선
- **호버**: 색상 변경 + 커서 변경 (resize cursor)
- **드래그 중**: 하이라이트 + 그림자 효과
- **최소 패인 크기**: 디바이더 드래그 시 최소 크기 제한

### 패인 렌더링

각 패인은 독립적인 wgpu 렌더 영역:

1. 패인 영역 계산 (트리에서 재귀적으로)
2. 각 패인에 대해 터미널 셀 그리드 렌더링
3. 활성 패인 테두리 강조
4. 비활성 패인 디밍 (선택적)

## 최적화 전략

- **독립 렌더 영역**: 각 패인이 자체 damage tracking을 가진다. 변경된 패인만 재렌더링하고 나머지는 이전 프레임을 유지한다.
- **비활성 패인 렌더링 빈도 감소**: 포커스되지 않은 패인은 낮은 빈도로 렌더링한다 (예: 15fps vs 60fps). 출력이 없으면 렌더링을 완전 중단한다.
- **레이아웃 캐싱**: 트리 구조가 변경되지 않으면 영역 계산 결과를 캐싱한다. 패인 분할/삭제 시에만 레이아웃을 재계산한다.
- **디바이더 드래그 최적화**: 드래그 중 실시간 리사이즈 대신 고스트 라인을 표시하고 놓을 때 적용하는 옵션을 제공한다. 저사양 환경에서 유용하다.
- **패인 수 제한**: 너무 많은 패인으로 인한 GPU 리소스 분산을 방지한다. 적절한 최대값(예: 16개)을 설정하고 초과 시 경고한다.

## 구현 가능 여부

| OS | 가능 여부 | 비고 |
|----|-----------|------|
| Windows | ✅ 가능 | GPU 렌더링 |
| macOS | ✅ 가능 | GPU 렌더링 |
| Linux | ✅ 가능 | GPU 렌더링 |

GPU 가속 네이티브 GUI이므로 cmux의 부드러운 분할/애니메이션을 그대로 재현할 수 있다.
