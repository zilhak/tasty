# 08. 세션 복원

## cmux 구현 방식

- JSON 스냅샷 자동 저장 (기본 8초 간격, 설정 가능)
- 워크스페이스/패인 레이아웃 복원
- 작업 디렉토리 복원
- 스크롤백 복원 (최대 4000줄)
- 브라우저 URL/히스토리 복원
- ANSI-safe 스크롤백 잘라내기

## 크로스 플랫폼 구현 방안

### 저장 형식

JSON 또는 MessagePack으로 세션 상태 직렬화:

```json
{
  "version": 1,
  "windows": [
    {
      "x": 100, "y": 50, "width": 1920, "height": 1080,
      "fullscreen": false
    }
  ],
  "workspaces": [
    {
      "name": "agent-1",
      "color": "#3b82f6",
      "layout": {
        "type": "vertical",
        "ratio": 0.5,
        "children": [
          { "type": "pane", "id": 1, "cwd": "/home/user/project" },
          { "type": "pane", "id": 2, "cwd": "/home/user/project" }
        ]
      }
    }
  ]
}
```

### 저장 경로

| OS | 경로 |
|----|------|
| **Linux** | `~/.local/share/tasty/session.json` |
| **macOS** | `~/Library/Application Support/tasty/session.json` |
| **Windows** | `%LOCALAPPDATA%\tasty\session.json` |

`directories` 크레이트로 OS별 경로를 추상화.

### GUI 전용 복원 항목

| 항목 | 설명 |
|------|------|
| 윈도우 위치/크기 | 각 OS 윈도우의 위치, 크기, 모니터 정보 |
| 전체화면 상태 | 전체화면 여부 |
| 사이드바 너비 | 사이드바 커스텀 너비 |
| 패인 분할 비율 | 정확한 분할 비율 (float) |
| 폰트 줌 레벨 | 사용자가 조정한 폰트 크기 |

### 스크롤백 복원

PTY 출력을 링 버퍼에 보관하고 세션 저장 시 함께 직렬화.
ANSI 이스케이프 시퀀스 경계를 고려하여 잘라내야 한다 (CSI 중간 잘림 방지).

### 복원 흐름

1. 시작 시 세션 파일 읽기
2. OS 윈도우 복원 (위치, 크기, 모니터)
3. 사이드바/레이아웃 복원
4. 각 워크스페이스/패인 레이아웃 재구성
5. 각 패인에서 저장된 cwd로 셸 시작
6. 스크롤백 버퍼 복원 (터미널 셀 그리드에 로드)

## 최적화 전략

- **증분 저장**: 전체 세션을 매번 직렬화하지 않고, 변경된 부분만 저장한다. 워크스페이스/패인 단위로 dirty flag를 관리한다.
- **스크롤백 압축**: 스크롤백 버퍼를 zstd 등으로 압축하여 디스크 사용량을 줄인다. 압축률과 속도의 균형을 위해 zstd level 3 정도를 사용한다.
- **비동기 저장**: 저장 작업을 별도 스레드에서 처리하여 UI 블로킹을 방지한다. 메인 스레드에서는 스냅샷만 찍고 I/O는 백그라운드에서 수행한다.
- **저장 디바운스**: 기본 8초 간격으로 자동 저장한다 (cmux와 동일). 설정에서 간격을 변경할 수 있다.
- **복원 속도**: 대용량 세션 복원 시 UI를 먼저 표시하고 스크롤백은 점진적으로 로드한다. 사용자가 즉시 입력을 시작할 수 있도록 한다.

| macOS | ✅ 가능 | 윈도우 위치 복원 포함 |
| Linux | ✅ 가능 | 윈도우 위치 복원 포함 |
